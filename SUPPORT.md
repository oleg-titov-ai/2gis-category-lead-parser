# Support

This repository is maintained as a portfolio and educational project.

## Getting Help

Before opening an issue:

1. read the README;
2. check the setup checklist;
3. run the demo mode;
4. run the test suite;
5. review existing documentation in `docs/`.

## Useful Commands

```bash
make test
make demo
```

## What to Include in an Issue

- short problem description;
- steps to reproduce;
- expected behavior;
- actual behavior;
- Python version;
- whether the problem happens in demo mode or live mode.

## What Not to Include

Do not include API keys, database passwords, `.env` files, private leads, production URLs, or real customer data.

## Project Status

This is not a commercial support channel. Issues and improvements are handled as time allows.
