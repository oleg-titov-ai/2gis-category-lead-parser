# Public Demo Notes

Use this note when preparing a public demo of the repository.

## Demo Goals

- Show the main workflow quickly.
- Keep the setup path easy to repeat.
- Use demo values only.
- Explain what the project does in business terms.
- Keep limitations visible.

## Before Sharing

- Check that the README is current.
- Check that commands still work.
- Check that examples are safe.
- Check that screenshots do not reveal private data.
