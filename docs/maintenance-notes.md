# Maintenance Notes

This file keeps lightweight project maintenance rules in one place.

## Regular Checks

- Run the unit test suite before changing parser logic.
- Keep demo mode working independently from live API access.
- Review dependency updates before merging.
- Keep public examples free from private lead data.
- Update the changelog when behavior changes.

## Safe Data Rules

Use placeholders for companies, contacts, phone numbers, email addresses, domains, API keys, webhook URLs, and database credentials.

## Release Hygiene

Before publishing a new portfolio update, check README accuracy, CI status, setup instructions, demo output, and security notes.
